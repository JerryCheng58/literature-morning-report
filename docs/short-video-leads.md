# 短视频引流文案

目标：把短视频流量导向 GitHub 开源仓库和付费内测名单。不要承诺“保证发一区文献”或“替你做科研判断”，主张应是“自动化、可追溯、可配置、节省时间”。

## 账号定位

一句话定位：

> 我把自己的博士文献晨报工作流开源了，任何专业都可以改成自己的版本。

## 15 秒短视频脚本

标题：

> 我把博士每天用的文献晨报工作流开源了

口播：

> 我读博期间最省时间的一个工作流，就是每天自动检索和我课题相关的新文献，然后发到邮箱。现在我把它做成了开源 Skill。你只要填自己的专业、研究方向、关键词和排除词，它就能生成你的专属文献晨报，还能导出 EndNote、Zotero、NoteExpress 能用的格式。链接我放主页，技术同学可以自部署，不想折腾的可以预约托管版内测。

屏幕画面：

1. 展示 `research_profile.yml`
2. 展示运行命令
3. 展示 HTML 晨报
4. 展示 RIS/BibTeX/ENW 导出文件

结尾 CTA：

> 评论“晨报”或去主页拿开源地址。

## 30 秒短视频脚本

标题：

> 不同专业都能用的科研文献晨报助手

口播：

> 之前我发过一个每天自动给我推送文献晨报的工作流，很多人问能不能照着做。现在我把它整理成了通用开源版本。它不是只适合某一个课题方向，你可以换成肿瘤、材料、心理学、教育学。核心是这个研究画像模板：研究方向、关键词、必须同时满足的逻辑、排除词、数据库、期刊门槛、邮箱和导出格式。它会检索 PubMed、Europe PMC、Semantic Scholar、OpenAlex、Crossref、arXiv；如果你有 Web of Science、Scopus、IEEE 的 API key，也可以接进去。每天最多推送最重要的几篇，不重复，并给你摘要概述和选题灵感。

屏幕画面：

1. 5 个不同专业 profile 示例
2. 数据库状态栏，展示已检索和 skipped 数据库
3. 文献条目
4. 收藏/导出概念图

结尾 CTA：

> 开源版适合会配置的同学，托管版我会先开放少量内测名额。

## 60 秒深度脚本

标题：

> 我怎样把一个博士自用工作流做成可开源产品

结构：

1. 痛点：每天刷数据库、去重、看摘要、整理到 EndNote 很耗时间。
2. 方案：用研究画像生成检索式，自动查数据库，去重，生成双语晨报。
3. 质量边界：只做可追溯来源，不编造 DOI、PMID、影响因子；商业数据库没权限就显示 skipped。
4. 适配：EndNote、Zotero、NoteExpress 导出。
5. 商业版：不想配置 key 和 SMTP 的用户，可以用托管版，邀请码内测。

口播结尾：

> 如果你是科研工作者，想把这个改成自己的方向，开源仓库在主页。想直接用托管版，评论“内测”。

## 置顶评论

> 开源地址：GitHub 搜索 `literature-morning-report`。  
> 自部署需要自己配置 API key 和邮箱。  
> 不想配置的同学可以预约托管版内测，我会先开少量邀请码。

## 私信回复模板

关键词“晨报”：

> 这是开源版地址：{GitHub 链接}。你可以先看 `README.md`，复制 `research_profile.template.yml`，改成自己的研究方向。

关键词“内测”：

> 托管版主要适合不想配置 API key 和邮箱的同学。第一批是邀请码制，每天推送专属文献晨报，支持收藏和导出。你可以发我：专业 + 研究方向 + 常用邮箱，我整理内测名单。

关键词“数据库”：

> 开源版默认支持 PubMed、Europe PMC、Semantic Scholar、OpenAlex、Crossref、arXiv。Web of Science、Scopus、Embase、IEEE Xplore 需要你自己或机构有 API 权限。
