## 推荐选题

主标题：

> 评论区都在求教程：上条 9.1 万播放的文献晨报工作流，我开源了

备选标题：

> 我把博士每天用的文献晨报工作流开源了，3 分钟教你改成自己的研究方向

推荐发第一条。它同时承接上一条爆点和评论区需求，能让老观众知道“教程兑现了”，也能让新观众通过 9.1 万播放建立信任。不建议先发网站会员版。当前阶段用户最关心的是“我能不能马上照着做”，开源教程更容易获得收藏和转发，会员邀请码放在评论区和结尾承接即可。

## 推荐视频结构

时长：60-90 秒。

形式：真人口播 + 屏幕录制。不建议纯口播，也不建议只放代码界面。

核心卖点顺序：

1. 痛点：每天刷数据库、去重、看摘要、导出 EndNote 很耗时间。
2. 结果：每天邮箱收到 5 篇以内的专属文献晨报。
3. 通用性：不是只适合我自己的具体课题方向，肿瘤、材料、心理学、教育学都可以填自己的画像。
4. 实操：展示 `research_profile.yml` 改研究方向、关键词、排除词、数据库、邮箱。
5. 可信边界：支持 PubMed、OpenAlex、Crossref 等；Web of Science、Scopus 需要机构 API 权限，不胡乱承诺。
6. 转化：开源版适合会配置的人；不想配置的人评论“内测”拿托管版邀请码。

## 60-90 秒口播稿

如果你是科研人，你应该很熟悉这个痛苦：每天打开 PubMed、Web of Science，换关键词，筛标题，看摘要，最后还要复制到 EndNote 或 Zotero。时间花了很多，但真正有启发的文献可能就几篇。

我最近把自己博士期间用的文献晨报工作流整理成了一个开源 Skill。它不是固定服务某一个课题，而是可以让你填自己的研究画像：专业、研究方向、核心关键词、必须同时满足的逻辑、排除词、数据库偏好、期刊门槛、邮箱和导出格式。

比如你如果是肿瘤方向，可以填肿瘤免疫、单细胞、类器官、药物筛选；如果是材料方向，可以填材料体系、制备方法和性能指标；如果是心理学或教育学，也可以换成对应的关键词、量表、实验设计和排除词。

它会自动检索 PubMed、Europe PMC、OpenAlex、Crossref、Semantic Scholar 等数据库；如果你有 Web of Science、Scopus、Embase 这些数据库的 API 权限，也可以接进去。最后生成一封中英文文献晨报，去重、摘要概述、相关性判断、科研灵感、EndNote 和 Zotero 能用的引用格式都会一起生成。

开源版我已经放到 GitHub。会配置的同学可以自己部署；不想配置 API key 和邮箱的，可以评论“内测”，我后面会开放少量托管版邀请码。

## 屏幕画面安排

1. 0-5 秒：邮箱晨报或 HTML 报告结果页，先给结果。
2. 5-20 秒：打开 `research_profile.yml`，突出“研究方向、关键词、排除词”。
3. 20-35 秒：展示不同专业都能改的字段，例如肿瘤、材料、心理学。
4. 35-50 秒：展示终端运行命令，不要停留太久。
5. 50-65 秒：展示生成的 HTML 晨报和 RIS/BibTeX/ENW 导出文件。
6. 65-90 秒：展示 GitHub README 和评论区 CTA。

## 流量预测

预测基础：上一条同主题视频已达到 9.1 万播放，并产生约 200 涨粉。下一条是该爆点的兑现教程，理论上转化质量更高，但自然播放可能低于第一条展示型视频，因为教程内容更窄。

单强样本校准播放区间：

- 低位：8000-30000 播放。说明教程太代码化，前 3 秒没有承接上一条。
- 中位：30000-80000 播放。说明承接了上一条兴趣，但教程天然比展示型内容窄。
- 高位：80000-180000 播放。条件是标题明确写“上条 9.1 万播放”，开头直接展示开源仓库和结果。
- 极高位：180000+ 播放。需要评论区出现大量“求链接”“求配置”“内测”“不会代码能不能用”。

涨粉预测：

- 低位：20-80 个粉丝。
- 中位：80-250 个粉丝。
- 高位：250-500 个粉丝。

转化预测：

- GitHub star：20-100
- 私信/评论“内测”：20-80
- 第一批高意向付费用户：5-20

这条视频的核心目标不应只看播放量，而应看收藏率、评论关键词和私信转化。对工具教程类内容，收藏和私信比泛播放更接近付费信号。

## 提升播放和涨粉的关键动作

标题必须让用户立刻知道“能拿走”：

> 评论区都在求教程：上条 9.1 万播放的文献晨报工作流，我开源了

前三秒不要讲背景，直接给结果：

> 上条视频评论区很多同学在求教程，今天我把这个文献晨报工作流整理成开源 Skill 了。

评论区置顶：

> 评论区求的教程来了。会配置的同学：GitHub 搜索 `literature-morning-report`，按 README 改 `research_profile.yml`。不想配置 API key、邮箱、定时任务的同学：评论 `内测`，我整理第一批托管版邀请码。

引导评论关键词：

- “教程”：自动回复开源地址和 README 路径。
- “内测”：收集托管版意向。
- “数据库”：解释 Web of Science、Scopus 需要 API 权限。
- “导出”：强调 EndNote/Zotero/NoteExpress 的 RIS/BibTeX/ENW 导出。

## 不建议这样拍

不要一开始讲“我要做一个产品”或“我要会员收费”。这会降低收藏动机。

不要把视频拍成纯代码教程。科研用户先关心结果，其次才关心怎么配置。

不要承诺“自动找到一区高分文献”或“替你筛好最有价值文献”。更稳妥的表达是“自动检索、去重、概述、提供可追溯来源和灵感提示”。

## 复盘指标

发布 24 小时后记录：

- 播放量
- 点赞数
- 收藏数
- 评论数
- 私信数
- 新增粉丝
- GitHub star / fork
- 评论关键词中“晨报 / 内测 / 数据库 / EndNote”的数量

判断标准：

- 收藏率高于点赞率的 50%：教程价值成立。
- “内测”评论或私信超过 20 个：可以启动邀请码承接。
- GitHub star 超过 20：开源获客路径成立。
- 大量问“不会配置怎么办”：托管版付费需求成立。
