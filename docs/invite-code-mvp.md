# 邀请码承接方案

第一版不要做复杂支付系统。建议先用第三方平台收款，再人工或半自动发邀请码。

## 用户流程

1. 用户从短视频进入 GitHub 或内测表单。
2. 用户选择自部署开源版或托管内测版。
3. 托管内测用户通过第三方平台付款。
4. 你给用户发邀请码。
5. 用户在网站输入邮箱和邀请码激活。
6. 用户填写研究画像，系统每日推送晨报。

## 会员档位

基础版：

- 1 个研究画像
- 每日 1 封晨报
- 每封最多 5 篇
- RIS/BibTeX/ENW/CSL-JSON 导出

专业版：

- 3 个研究画像
- 收藏夹
- 历史晨报
- 优先模板更新

课题组版：

- 多用户
- 统一模板
- 课题组共享收藏夹
- 定制关键词和排除词

## 运营指标

两周验证目标：

- 100 个意向用户
- 20 个付费会员
- 次日打开率超过 40%
- 7 日留存超过 30%
- 每周收集至少 10 条有效反馈

## 风险边界

- 不承诺替代人工文献筛选。
- 不承诺所有数据库都能访问。
- 不承诺 Web of Science/Scopus/Embase 覆盖，除非账号实际具备 API 权限。
- 不发送完整受版权保护的摘要或全文，只发送概述、链接和可核验元数据。
